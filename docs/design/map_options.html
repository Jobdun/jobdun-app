Jobdun · Top 5 Map Experiences

What's there today: flutter_map + raster tiles, identical generic pins, tap-a-pin → jump straight to the detail page. That's the 2015 pattern — no preview before commitment, no information in the pin, no clustering, nothing in the thumb zone. The modern bar (Airbnb / Zillow / Google Maps 2025-26): the pin carries information, selection previews before navigating, and browsing happens in the bottom third of the screen.

Verdict

Ship #1 + #2 together as one slice — price-bearing pins with clustering, synced to a bottom card carousel. That's the full modern marketplace map (Airbnb mechanics, Zillow pins) and it needs no new packages, just flutter_map marker widgets + a PageView. #4 Service-Radius Dial layers on top afterwards as the genuinely differentiated tradie feature no competitor has. #3 is the long-term destination if the map becomes the primary browse surface; #5 is the honest fallback if it doesn't. One data prerequisite for all of them: real distance/geo on jobs (the feed currently has none — we removed the fake "0.0 km" rather than lie).

Research: Airbnb/Zillow map-list sync patterns · NN/g thumb-zone (carousel/chips bottom, not top) · marker-clustering UX (Google Maps Platform guidance) · pin-information density (Zillow/Domain price pins) · current code: flutter_map 7, generic Markers → direct detail push (home_map_view.dart).
